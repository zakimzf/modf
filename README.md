# Binance Volatility Monitor (Supabase Edition)

This service monitors the Binance WebSocket API for real-time ticker data, sends alerts for significant price volatility, and provides scheduled reports on the top gaining assets, all powered by a Python service on a VPS and Supabase for the backend.

## Features

- Real-time volatility alerts (±3% within a 1-minute window).
- Scheduled reports of top 5 gainers at 1m, 5m, 10m, and 1h intervals.
- Data storage in Supabase (PostgreSQL).
- Email notifications via a Supabase Edge Function (using Resend).

## Requirements

- A Linux-based VPS (e.g., Ubuntu 20.04 or later).
- Python 3.8 or higher.
- A Supabase account.
- A Resend account and API key for email sending.
- Supabase CLI installed on your local machine (for deploying the Edge Function).

## Deployment Instructions

### Part 1: Supabase Setup

#### 1. Create the Database Table

In your Supabase project, go to the SQL Editor and run the following query to create the `price_history` table:

```sql
CREATE TABLE public.price_history (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  symbol TEXT NOT NULL,
  price DOUBLE PRECISION NOT NULL,
  timestamp TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Create an index on the timestamp and symbol for faster queries
CREATE INDEX idx_price_history_timestamp_symbol ON public.price_history (timestamp DESC, symbol);

-- Supabase API can interact with the table
ALTER TABLE public.price_history ENABLE ROW LEVEL SECURITY;
```

#### 2. Deploy the Edge Function

The code for the `send-email` Edge Function is located in the `supabase_functions` directory. To deploy it, you will need the Supabase CLI.

From the `binance_monitor` directory on your local machine:

```bash
# Link your local project to your Supabase project
supabase link --project-ref <your-project-id>
# Deploy the Edge Function
supabase functions deploy send-email
```

#### 3. Add Secrets to Supabase

In your Supabase project dashboard, go to `Project Settings` -> `Edge Functions` and add a new secret:
- **Name:** `RESEND_API_KEY`
- **Value:** Your API key from Resend.

### Part 2: VPS Setup

#### 1. Clone the Repository

Clone this repository to your VPS:

```bash
git clone <your-repo-url>
cd binance_monitor
```

#### 2. Set up Python Environment

```bash
sudo apt update
sudo apt install python3-venv -y
python3 -m venv venv
source venv/bin/activate
```

#### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

#### 4. Configure the Application

Create a `.env` file from the example template:

```bash
cp .env.example .env
```

Now, edit the `.env` file with your specific settings:

```bash
nano .env
```

Update the following fields:
- `SUPABASE_URL`: Your Supabase project URL.
- `SUPABASE_KEY`: Your Supabase `anon` key.
- `FROM_EMAIL`: (Optional) The email address you want to send notifications from.
- `TO_EMAIL`: (Optional) The email address where you want to receive notifications.
- `TELEGRAM_BOT_TOKEN`: (Optional) Your Telegram bot token.
- `TELEGRAM_CHAT_ID`: (Optional) The chat ID for your Telegram channel or user.

#### How to Get a Telegram Bot Token and Chat ID

1.  **Create a Bot:** Talk to the [BotFather](https://t.me/botfather) on Telegram. Send the `/newbot` command and follow the instructions. The BotFather will give you a token.
2.  **Get Your Chat ID:**
    *   **For a private chat:** Send a message to your bot, then visit `https://api.telegram.org/bot<YourBOTToken>/getUpdates`. You will see your chat ID in the response.
    *   **For a channel:** Add your bot to the channel as an administrator. Send a message to the channel, then visit the same URL as above to find the channel's chat ID (it will start with `-`).

### Part 3: Running the Service

Follow the same steps as before to set up and start the `systemd` service. Ensure you have updated the `deployment/binance-monitor.service` file with the correct paths for your environment.

```bash
# Edit the service file with your paths
nano deployment/binance-monitor.service

# Copy the service file
sudo cp deployment/binance-monitor.service /etc/systemd/system/binance-monitor.service

# Reload, start, and enable the service
sudo systemctl daemon-reload
sudo systemctl start binance-monitor
sudo systemctl enable binance-monitor

# Check the status and logs
sudo systemctl status binance-monitor
sudo journalctl -u binance-monitor -f
```

## Testing

This project uses `pytest` for unit testing. The tests are located in the `tests/` directory and use mocks to isolate the application logic from external services like Supabase.

To run the tests, first make sure you have installed the testing dependencies:

```bash
# Activate your virtual environment if you haven't already
source venv/bin/activate
# Install all dependencies, including for testing
pip install -r requirements.txt
```

Then, you can run the test suite from the `binance_monitor` directory:

```bash
pytest
```

The tests will run and provide a summary of the results. This is a great way to verify that the core logic is working correctly before deploying the application.
